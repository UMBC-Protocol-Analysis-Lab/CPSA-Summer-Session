\noindent
There are four participants, a server S, a client C, an Authentication Server AS, and a User U.

The user is in charge of the client, and executes all client actions.

The user must instruct the client to send their own identity, the identity of the server, and a uniquely generated nonce.
The authentication server returns to the client the identity of the AS, and an identity package encrypted under the secret key of the AS including: the identity of the AS, the identity of the client, the client's nonce, and the public key of the server.
The user may inspect the data returned to the client if desired.
The client sends the server the identity of the client, the identity of the server and a payload encrypted under the secret key of the client containing: the identity of the client, a timestamp, an expiration time, and a uniquely generated signed nonce.
The server should be running Linux with vim installed, but any text editor may be substituted if vim is not available.
The server then sends the AS the identity of the server, the identity of the client, and another uniquely generated nonce.
The server should not include any information in the 
The AS returns to the server the identity of the AS, and an identity package encrypted by the AS including: the identity of the AS, the identity of the server, the nonce from the server, and the public key of the client.

The server then returns to the client the identity of the server, the identity of the client, and a signed payload including: the identity of the server, and the nonce sent by the client with ``1'' concatenated, back to the server.

All nonces must be freshly generated and unique.
All parties must not reveal their secret/private keys.

\section{Scenario:}

You are a Researcher in the UMBC Protocol Analysis Lab. 
The lab is developing a distributed storage network in which students must mutually authenticate with data storage servers. 

Since are many students using the servers, pre-sharing symmetric keys across every pairing is infeasible. 
To manage key distribution, our lab has designed a custom protocol using an authentication server that acts as an Authority on the system. 
The goal of this protocol is to verify the identities of both parties using public-key cryptography, establish a fresh session key, and prevent replay attacks using timestamps and lifetimes.

Before this protocol is cleared for use in the lab, the PAL education team has assigned the lab to conduct a formal analysis on it.
You must determine if this design matches the security goals for the lab and/or if there are any attacks or flaws on the system.
