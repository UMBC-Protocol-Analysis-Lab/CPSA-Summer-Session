\begin{frame}\frametitle{What is the problem space?}
\pause
We start with a simple problem.

\pause
\isred{Problem:} Alice and Bob want to set up a meeting without Charlie finding out, how do they do it?

\pause
Can you think of some na\"ive solutions?

\pause
\begin{enumerate}
\item Alice can tell Bob about the meeting in another language, one that Charlie doesn't know?
\pause
\item Alice can hide information about the meeting in random places that Bob knows to check?
\pause
\item Alice and Bob can go somewhere that they're sure that no one is around, and then talk about the meeting?
\pause
\item Other ideas?
\end{enumerate}
\end{frame}

\begin{frame}\frametitle{Can we standardize obfuscation methods?}
\pause
How can we turn these ideas into concrete concepts.
\pause
Eventually -- turn these ideas into mathematical statements.

\pause
A language requires both the sender and receiver to have a ``key'', that lets them understand how words correspond to things.

\pause
Hiding information corresponds to presharing information, and using a combination of presharing and given information, Bob can derive information.

\pause
Alice and Bob making sure no one is around is like an airgap.
\pause
Can they really be sure \emph{no one} is listening?
\end{frame}

\begin{frame}\frametitle{What is a cryptographic primitive?}

\begin{definition}
A cryptographic primitive is a foundational operation in cryptography.
All cryptographic algorithms are constructed from cryptographic primitives.
\end{definition}

\pause
We consider several operations that commonly appear.

\begin{frame}\frametitle{CPSA regular strand actions}

\begin{columns}
\column{0.6\textwidth}
\pause
\begin{enumerate}
\item[M.] Text Message: \( \langle +t \rangle \) where \(t \in T\)
\pause
\item[T.] Tee: \( \langle -g, +g, +g \rangle \) where \(g \in G\)
\pause
\item[C.] Concatenation: \( \langle -g, -h, +gh \rangle \) where \(g,h, gh \in G\)
\pause
\item[S.] Separation into components: \( \langle -gh, +g, +h \rangle \)
\pause
\item[K.] Key: \( \langle +K \rangle \) where \(K \in \mathbf{K_p}\)
\pause
\item[E.] Encryption \( \langle -K, -h, +\{h\}_K \rangle \)
\pause
\item[D.] Decryption \( \langle -K^{-1}, -\{h\}_K, +h \rangle \)
\pause
\item[H.] Hash \( \langle -g, +hash(g) \rangle \)
\end{enumerate}

\column{0.4\textwidth}
\pause
\begin{enumerate}
\item \(\langle +m \rangle\) emit a message from the network.
\pause
\item \(\langle -m \rangle\) extract a message from the network.
\pause
\item \(\langle m_1m_2 \rangle\) concatenate two messages.
\pause
\item \(\langle \{m\}_k \rangle\) encrypt a message \(m\) under key \(K\).
\end{enumerate}
\end{columns}
\end{frame}

\item[S.] Separation into components: \( \langle -gh, +g, +h \rangle \)

\item[K.] Key: \( \langle +K \rangle \) where \(K \in \mathbf{K_p}\)

\item[E.] Encryption \( \langle -K, -h, +\{h\}_K \rangle \)

\end{enumerate}

\begin{frame}\frametitle{Symmetric encryption}
\pause
Let \(m\) be a message, \(k\) be a \emph{secret} key, and \(c\) be ciphertext.
\pause
\begin{definition}[Encryption]
A function \(e\) is called an \emph{encryption function} if:
\[
e(m,k) = c
\]
\end{definition}
\pause
\begin{definition}[Decryption]
A function \(d\) is called a \emph{decryption function} if:
\[
d(c,k) = m
\]
\end{definition}
\end{frame}

\begin{frame}\frametitle{Asymmetric encryption}
\pause
Let \(m\) be a message, \(pub_k,priv_k\) be a public-private key pair, and \(c\) be ciphertext.
\pause
\begin{definition}[Encryption]
A function \(f\) is \emph{encrypting} if:
\[
f(m,pub_k) = c
\]
\end{definition}
\pause
\begin{definition}[Decryption]
A function \(f\) is \emph{decrypting} if:
\[
f(c,priv_k) = m
\]
\end{definition}
\end{frame}

\begin{frame}\frametitle{Signing}
\pause
Let \(m\) be a message, \(pub_k,priv_k\) be a public-private key pair, and \(c\) be ciphertext.
\pause
\begin{definition}[Signing]
A function \(f\) produces a \emph{signature} if:
\[
f(m,priv_k) = c
\]
\end{definition}
\pause
\begin{definition}[Verifying]
A function \(f\) can be used to \emph{verify} if:
\[
f(c,pub_k) = m
\]
\end{definition}
\end{frame}

\begin{frame}\frametitle{Hashing}
\pause
Let \(m\) be a message, \(md\) be a message digest.
\pause
\begin{definition}[Hashing]
A function \(h\) is called an \emph{hash function} if:
\begin{enumerate}
\item \(h(m) = md\)
\item It is impractical to determine \(m\) given \(md\).
\item Collision Resistance: It is infeasible to find \(m_1,m_2\) such that \(h(m_1) = h(m_2)\).
\end{enumerate}
\end{definition}
\end{frame}
